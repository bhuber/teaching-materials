Shells

A text-based command interpreter.

The most common shell is "bash".

On Mac, use a shell via the "Terminal" app.

Shell prompt

The prompt usually shows you your username and computer name:

        username@hostname:

To see what directory you're in, use the pwd command: 

        username@hostname: pwd
        /Users/username/Desktop

Run Away! Run Away!

To exit the shell, simply type "exit". Now start it up again!

Type sleep 5. Not too bad, only wasted five seconds of your valuable time. Now type sleep 500

Use CTRL-C to kill whatever is currently running if it's taking too long or it looks at you funny. Use it now to kill the sleep command.

File Paths

A sequence of nested directories with a file or directory at the end, separated by "/":

  • Relative - pretend whatever pwd prints comes in front: documents/fun/microsoft_jokes.html
  • Absolute - always starts at the top, or "Root" of the file system: /usr/bin/local
  • Root: /

Shortcuts:

  • Current directory: .
  • Parent directory: ..
  • Home directory: ~

Change directory

Use the cd command to change directories:

        cd mydirectory/
        
        cd ../

List directory

Use the ls command to list the contents of a directory:

        username@hostname: ls
        mydir/
        myfile.txt

For more details, use -l, and to see hidden files, use : 

        username@hostname: ls -al
        -rw-r--r--    16 bhuber  staff      544 19 Mar 11:40 .dropbox
        drwxr-xr-x    16 bhuber  staff      544 19 Mar 11:40 mydir/
        -rw-r--r--    16 bhuber  staff      544 19 Mar 11:40 myfile.txt

File Names

  • Case sensitive
  • Any character (besides "/")
  • File extensions are for user convenience
  • Files with special characters (e.g. whitespace) can be put inside quotes. Or just don't make file names with whitespace in them.

Examples: 

        README   .bashrc   index.html    index.html.old    

        "Bugs List.txt"

Making and deleting directories

To make a new directory, use the mkdir command:

        mkdir myproject

To remove a (empty) directory, use the rmdir command:

        rmdir myproject

Filesystem structure

Varies from system, but some common folders:

  • /bin: Basic commands
  • /etc: System config files
  • /tmp: Temporary files
  • /usr: Regular user tools
  • /usr/local: Installed user software
  • /var: Data used by system

Working with files

To make a file, use the touch command:

touch newfile.txt

To move or copy a file, use the mv and cp commands:

cp newfile.txt newerfile.txt

To remove a file, use the rm command:

rm newerfile.txt

Save yourself some typing

  • The left/right arrow keys let you edit within a command. CTRL-f and CTRL-b also work (forward/back).
  • The up/down arrow keys let you select previous commands. CTRL-p and CTRL-n also work (previous/next).
  • CTRL-a goes to beginning of the line, CTRL-e to the end.
  • TAB auto-completes filenames. Hit it twice for multiple results.
  • DELETE or CTRL-d deletes the character to the right of the cursor

Command Shortcuts

use history to see your previous commands:

          564  grep 'jquery' *.txt > results.txt
          565  vi results.txt 
          566* open -a textedit myfile.txt
          568  history

also useful:

  • !!: last command
  • CTRL-r: search command history

Editing files

You can use various built-in editors that typically appear on *nix systems:

  • vi myfile.txt
  • emacs myfile.txt
  • nano myfile.txt

If you're not familiar with vi or emacs use nano for now. Note all the keyboard shortcuts are at the bottom of the screen and ^ stands for Control.

Viewing files

You can view files in an editor, or for read-only viewing, try:

  • cat myfile.txt
  • less myfile.txt

You can pass multiple files into any of them:

cat myfile.txt myfile2.txt myfile3.txt

Less is more

less is a very useful tool for examining large blocks of text. Run the following commands:

       cd /etc
       ls -al

Now try ls -al | less

Less is more (continued)

less is pretty simple and has mostly one-key navigation commands

  • h gives a list of them
  • q quits
  • u and d scroll up and down (Page Up, Page Down, Space, and Arrow keys work too)
  • / is like CTRL-f in a web browser: you can search for text. n and p cycle through the results
  • g and G go to the beginning and end of the file respectively

HALP!

There are two primary sources for commands run in bash: help and man. Explore them using the following commands (don't bother reading all the text that appears): 

        help
        help cd
        help help
        man ls
        man man
        man which
        which ls
        man -k ssh

Searching files

Use the grep command to search in files:

Find the phrase "hello" in a specific file:

grep hello myfile.txt

Find the phrase "hello" in all txt files:

grep hello *.txt

Use man grep to get more info on how grep works

Finding files

Use the find command to find files according to name/metadata:

Find all txt files under the current directory:

        find . -name '*.txt' -print

Use man find to get more info on how find works

Redirecting output

By default, input comes from the screen and output goes to the screen.

You can use < and > to redirect input/output.

Save results of search in a file:

        grep hello *.txt > results.txt
        less results.txt
        echo "Hello World"
        echo "Hello World" > hello_world.txt
        less hello_world.txt

Users and Groups

  • *nix systems almost always have multiple users
  • Groups are collections of users. Users can belong to multiple groups, and groups can have multiple users. The id command will tell you which groups you belong to. Use man -k group to find out more.
  • There is always a special user called "root". This is the administrative or super user. Root is allowed to do pretty much anything
  • All processes - including your terminal - have a user associated with them (this is what the whoami command displays)

Changing your User

  • There are two main commands for running things as other users: su (switch user) and sudo (switch user - do)
  • su myusername starts a new terminal running as myusername (provided you know the password for that user). su alone gives you a terminal as root if you know the root password. Type exit to get back to your original terminal.
  • sudo ... runs whatever comes after it as root, provided you're allowed to use the sudo command (how this is determined varies based on the system). sudo prompts you for YOUR password (note this can be different than the root password).
  • Use sudo su to get a terminal as root; use whoami to verify it worked.
  • Be careful running things as root, *nix will generally let you do very dumb things as root without complaint (e.g. deleting the contents of /bin, removing your user from the system)

Access Permissions

The default output for ls -l looks something like this:

File type and access permissions Link count (not important) Owning User Owning Group Size in bytes Last modified date Name
-rw-rw-r-- 1 phaedrus phaedrus 27314 May 12 21:52 slides.html

Every file on the system belongs to a user and a group, and has a set of access permissions which the system uses to enforce what actions different users can do.

Access Permissions (continued)

File Type User permissions Group permissions World permissions
- rw- rw- r--
  • There are several different possible values for file types, but the only two we care about are "-" for normal files and "d" for directories (note the . and .. entries in ls -al are always of type "d"
  • There are three types of permissions: read (r), write (w), execute (x). These mean different things for files vs directories.
  • There are three sets of rwx permissions for each file: User, Group, and World. User permissions apply if you're the file's owner, group permissions apply if you belong to the owning group, and world permissions apply otherwise.
  • Permissions are NOT inherited

Changing Access Permissions

  • There are three main commands for changing file access permissions: chown (change owner), chgrp (change group), and chmod (change mode)
  • chown someuser somefile - Changes a file's owner
  • chgrp somegroup somefile - Changes a file's group
  • chmod [u | g | o | a][+ | -][r | w | x] somefile - Add/remove permissions for somefile. u is for user (the file's owner), g is for group, o is for everyone else, a is for all. + adds the permission, - removes it.
  • Example: chmod g-x somefile removes the group execute permission for somefile, chmod a+r somefile makes somefile readable by everyone.

Exercise

  • Use sudo and/or su to make a file owned by root
  • Make the file only readable by the root user and nobody else
  • Try to read the file as your regular user
  • Change the file's owner and group to your regular user (same owner/group as the . directory)
